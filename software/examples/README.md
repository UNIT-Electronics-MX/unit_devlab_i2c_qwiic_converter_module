# Software Examples

In this example, the microcontroller generates a slow square wave on **GPIO6**. Since the microcontroller operates at **3.3 V logic**, the output signal is originally a **3.3 V digital pulse train**.

The signal is then connected to the **I2C QWIIC Converter Module**, which works as a logic level converter. In this configuration, the module converts the input signal from **3.3 V to 5 V**, allowing the square wave generated by the microcontroller to be observed or used as a **5 V logic signal** at the module output.

This test is useful to verify the module’s level-shifting behavior and to confirm that a low-voltage digital signal from a 3.3 V microcontroller can be safely adapted for circuits or devices that require 5 V logic levels.

---

## Arduino C++ Version

```cpp
const int pulsePin = 6;      // GPIO6
const int highTime = 2000;   // HIGH time: 2000 ms = 2 seconds
const int lowTime = 2000;    // LOW time: 2000 ms = 2 seconds

void setup() {
  pinMode(pulsePin, OUTPUT); // Configure GPIO6 as an output
}

void loop() {
  digitalWrite(pulsePin, HIGH); // Set GPIO6 HIGH
  delay(highTime);              // Wait 2 seconds

  digitalWrite(pulsePin, LOW);  // Set GPIO6 LOW
  delay(lowTime);               // Wait 2 seconds
}
```

---

## Arduino C++ Explanation

This program generates a slow digital pulse signal on **GPIO6**.

```cpp
const int pulsePin = 6;
```

This line defines the output pin. In this case, the pulse train will be generated on **GPIO6**.

```cpp
const int highTime = 2000;
const int lowTime = 2000;
```

These constants define how long the signal stays in each state. In Arduino, the `delay()` function uses milliseconds, so `2000 ms` is equal to `2 seconds`.

```cpp
pinMode(pulsePin, OUTPUT);
```

This line configures GPIO6 as a digital output pin.

```cpp
digitalWrite(pulsePin, HIGH);
```

This sets GPIO6 to a HIGH logic level. Depending on the board, this is usually **3.3 V** or **5 V**.

```cpp
delay(highTime);
```

This keeps the pin HIGH for 2 seconds.

```cpp
digitalWrite(pulsePin, LOW);
```

This sets GPIO6 to a LOW logic level, usually **0 V**.

```cpp
delay(lowTime);
```

This keeps the pin LOW for 2 seconds.

Since the code is inside `loop()`, the sequence repeats forever.

## MicroPython Version

```python
from machine import Pin
from time import sleep

pulse_pin = Pin(6, Pin.OUT)  # Configure GPIO6 as an output

high_time = 2  # HIGH time: 2 seconds
low_time = 2   # LOW time: 2 seconds

while True:
    pulse_pin.value(1)  # Set GPIO6 HIGH
    sleep(high_time)    # Wait 2 seconds

    pulse_pin.value(0)  # Set GPIO6 LOW
    sleep(low_time)     # Wait 2 seconds
```

---

## Signal Behavior

Both programs generate the following signal:

```text
HIGH for 2 seconds
LOW for 2 seconds
HIGH for 2 seconds
LOW for 2 seconds
...
```

The total period of the signal is:

```text
Period = HIGH time + LOW time
Period = 2 s + 2 s
Period = 4 s
```

The frequency is:

```text
Frequency = 1 / Period
Frequency = 1 / 4 s
Frequency = 0.25 Hz
```

The duty cycle is:

```text
Duty Cycle = HIGH time / Period × 100
Duty Cycle = 2 s / 4 s × 100
Duty Cycle = 50%
```
